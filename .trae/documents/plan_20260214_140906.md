# 执行方案：构建信息技术点名系统 (单表/学生颗粒度)

## 1. 后端实现 (Backend)

### 1.1 数据库模型
创建 `backend/app/models/xxjs/dianming.py`，定义 `XxjsDianming` 模型：
- `id` (PK)
- `year` (年份/届别)
- `class_name` (班级名称)
- `student_name` (学生姓名)
- `student_no` (学号，可选)
- `created_at`

### 1.2 API 接口
创建 `backend/app/api/endpoints/xxjs/dianming.py`，提供以下接口：
- `GET /api/v1/xxjs/dianming/classes` - 获取所有班级列表 (Distinct Year+Class)
- `GET /api/v1/xxjs/dianming/students` - 获取指定班级的学生列表
- `POST /api/v1/xxjs/dianming/import` - 批量导入学生 (接受 JSON 列表)
- `DELETE /api/v1/xxjs/dianming/class` - 删除整个班级的数据

注册路由到 `backend/app/api/__init__.py`，前缀为 `/xxjs`。

## 2. 前端实现 (Frontend)

### 2.1 API 服务
创建 `frontend/src/services/xxjs/dianming.ts`，封装上述接口调用。

### 2.2 界面重构
修改 `frontend/src/pages/Admin/ITTechnology/index.tsx`，实现：
- **入口网格**：展示“随机点名”等工具入口。
- **点名管理组件** (`DianmingManager`)：
    - 展示班级列表。
    - “新建/导入”弹窗：输入年份、班级，粘贴名单文本，自动解析并调用导入接口。
- **点名播放组件** (`RollCallModal`)：
    - 全屏/大窗口模式。
    - 加载该班级所有学生。
    - 实现随机滚动动画和抽取逻辑。

## 3. 执行顺序
1.  创建后端模型与更新 `__init__.py`。
2.  创建后端 API 路由与逻辑。
3.  注册后端路由。
4.  创建前端 API 服务文件。
5.  开发前端界面组件。
6.  验证功能。

## 一、项目背景与目标

### 1.1 教学闭环

```
上课（AI 智能体辅助学习）
  → 讨论（小组协作探究）
    → 自主检测（AI 出题，学生自我评价）
      → 画像（两级画像体系）
        → 初级画像：做完题自动生成（仅基于测评数据）
        → 高级画像：三维融合分析（测评 + 讨论 + 智能体）
```

### 1.2 核心价值

- 教师：实时掌握学生学习状态，精准定位知识薄弱点，数据驱动教学调整
- 学生：个性化自主检测（每人题目不同），即时反馈，可查看初级画像和三维画像
- 系统：三大数据源（智能体对话 + 小组讨论 + 自主检测）融合，生成多维画像

### 1.3 与原 v1.0 的关键区别

| 维度 | v1.0（课堂测评） | v2.0（自主检测） |
|------|-----------------|-----------------|
| 定位 | 考试性质，偏严肃 | 学生自主检测，轻量自省 |
| 画像 | 单一画像（教师触发） | 两级画像：初级（自动）+ 高级（教师触发） |
| 学生可见 | 只能看测评结果 | 可看初级画像 + 三维画像（两个按钮） |
| 融合分析入口 | 仅小组讨论深度分析弹窗 | 独立管理页面 `/admin/assessment` |

### 1.4 三大数据源

| 数据源 | 表前缀 | 核心数据 |
|--------|--------|----------|
| AI 智能体对话 | `znt_conversations` | 学生提问内容、AI 回答、响应时间 |
| 小组讨论 | `znt_group_discussion_*` | 讨论消息、成员参与、讨论分析 |
| 自主检测（新增） | `znt_assessment_*` | 题目、答题记录、得分、AI 评语 |

### 两级画像体系

| 级别 | 触发方式 | 数据来源 | 谁能看 |
|------|---------|---------|--------|
| 初级画像 | 学生提交测评后自动生成 | 仅本次测评数据（总分、各知识点得分、错题分析） | 学生本人 + 教师 |
| 高级画像（三维融合） | 教师在后台手动/批量触发 | 测评 + 小组讨论 + 智能体对话 | 学生本人 + 教师 |

### 第 1 期：后端基础 + 管理端

1. 创建 7 张数据表（Model + Alembic 迁移）→ 详见 [ASSESSMENT_DATABASE.md](./ASSESSMENT_DATABASE.md)

**验证：** 管理端创建测评 → AI 生成题目 → 审核编辑 → 开启测评

### 第 2 期：学生端 + 评分 + 初级画像

**当前状态：已完成**

1. 学生端浮动窗口（AssessmentPanel.tsx）→ 详见 [ASSESSMENT_FRONTEND.md](./ASSESSMENT_FRONTEND.md)
2. 开始测评 API（创建 session + 抽题逻辑）

4. 提交整卷（AI 评分填空/简答 + 生成初级画像）

**验证：** 学生打开浮动窗口 → 开始检测 → 答题 → 提交 → 自动看到初级画像

2. 管理端画像能力完善（当前并入 `StatisticsPage`，后续可拆分为独立画像中心页）
3. 学生端"查看三维画像"按钮

**验证：** 教师生成三维画像 → 学生可查看 → 个人/小组/群体三种画像

## 五、注意事项

1. **题目随机性**：每个学生的题目从题库中随机抽取，确保不同学生拿到不同的题目组合
2. **实时生成模式**：如果题库为空或教师选择实时生成，则在学生开始测评时调用 AI 实时出题，题目存入 `question_snapshot` 字段
3. **防作弊**：同一学生对同一测评只能有一个进行中的会话；答题记录不可修改
4. **数据关联**：通过 `sys_users.id` 关联三大数据源；通过 `znt_assessment_config_agents` 关联测评与课堂智能体
5. **画像缓存**：画像生成耗时较长（需调用 AI），生成后存入数据库，可重复查看
6. **评分一致性**：选择题由系统精确判分；填空/简答题由 AI 评分，评分 prompt 需要包含参考答案和评分标准
7. **初级画像自动生成**：学生提交测评后，后端自动调用 AI 生成初级画像，无需教师干预
8. **学生画像可见性**：学生可查看自己的初级画像和三维画像，但不能查看其他学生的画像

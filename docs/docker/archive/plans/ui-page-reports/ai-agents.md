## 2) 问题分级（P0-P3）

- P0（比例失衡）：移动端侧栏高度使用固定 `calc(100%-56px)`，在不同头部高度下会出现内容裁切风险；部分浮层固定宽高导致小屏拥挤。
- P1（字号体系冲突）：侧栏状态 Badge、对话区工作流标签和浮层标签存在 `text-[10px]/text-[11px]` 局部缩小，与正文档位冲突。
- P2（间距密度不一致）：聊天头/消息/输入区和三个浮层（讨论/测评/课堂）的局部尺寸档位不统一，存在“头部紧、内容松”现象。
- P3（细节噪声）：同页仍有多个局部硬编码尺寸（`w-[140px]`、`w-[420px]`、`min-w-[48px]` 等），维护成本高。

## 3) 规则映射（违反项 -> 规则）

- 局部小字号 Badge（10/11） -> 违反“排版规则（辅助 12-14，禁止双体系）”
- 浮层与聊天区密度不一致 -> 违反“间距规则（固定档位）+ 骨架比例规则（聊天密度统一）”
- 侧栏/浮层局部硬编码宽高 -> 违反“骨架比例规则 + 响应式规则”

## 4) 本页改动边界（单页最小改动）

- 只改：`/ai-agents` 页面及其子组件的布局尺寸、字号、间距、浮层样式 token 化与响应式收敛。
- 不改：智能体切换、会话历史、流式消息、小组讨论/测评/课堂互动的业务逻辑与交互路径。

# 单页式 UI 治理基线（v1）

更新时间：2026-04-04

## 1. 目标与执行原则

- 目标：先统一规则，再按“**一页一轮**”收敛，避免跨页大修造成比例失衡与回归。
- 原则：单页最小改动，只改样式与布局，不改业务逻辑/接口/权限。
- 验收：每页都必须完成静态检查、视觉检查、交互回归，再进入下一页。

## 2. 统一规则（必须遵守）

### 2.1 排版规则
- 导航文字：14-16
- 页面标题：18-22
- 正文：14-16
- 辅助信息：12-14
- 禁止同屏出现明显冲突的双字号体系。

### 2.2 间距规则
- 组件内边距仅使用固定档位（S/M/L）。
- 区块间距保持同级一致，禁止局部“忽大忽小”。

### 2.3 骨架比例规则
- 顶部导航高度使用统一 token，不允许页面内私自放大/缩小。
- 侧栏与主区比例优先控制在 22%-28% : 72%-78%。
- 聊天场景统一“头部-消息区-输入区”密度，消息字体不得小于正文档。

### 2.4 组件外观规则
- 卡片：统一圆角、边框层级、背景层级。
- 按钮：同级操作统一尺寸与字重。
- 浮动入口：仅统一尺寸/圆角/阴影/文字层级，不改交互路径。

### 2.5 响应式规则
- 1366 / 1536 / 1920 / 375 保持同一视觉语言。
- 小屏优先保证可读性，再压缩间距，不先压缩字体。

## 3. 页面诊断标准（固定格式）

- 当前问题：按 P0-P3 分级。
- 规则映射：每个问题必须映射到具体规则条目。
- 改动边界：明确“只改什么、不改什么”。

问题分级定义：
- P0：比例失衡（导航压内容、左挤右空、头重脚轻）
- P1：字号体系冲突（同屏两套文字尺度）
- P2：间距/卡片密度不一致（胖瘦混杂）
- P3：细节噪声（阴影、图标、按钮风格不统一）

## 4. 单页推进流程（严格）

1. 指定下一页（单一路由）。
2. 输出该页体检报告（P0-P3 + 规则映射 + 改动边界）。
3. 提交该页最小改动方案（仅样式布局）。
4. 完成该页专项验收：
   - `npm run type-check`
   - 4 分辨率视觉检查（1366/1536/1920/375）
   - 关键交互回归
5. 页面通过后，更新台账并进入下一页。

## 5. 配套工具

- 全局审计：`npm run ui:audit`
- 审计门禁：`npm run ui:audit:ci`
- 单页审计（按路由）：`npm run ui:audit -- --route /ai-agents`
- 单页体检报告生成：`npm run ui:page:report -- --route /ai-agents`
- 单页体检 + type-check：`npm run ui:page:verify -- --route /ai-agents`

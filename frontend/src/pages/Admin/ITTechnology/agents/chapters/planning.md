# 任务规划：把大事化小，一步步来

你有没有过这样的经历——周末面对一堆作业，不知道从哪开始，于是刷了一上午手机？

这就是缺了一个**计划**。AI Agent 也一样——面对复杂任务，它需要一个"计划器"来把大事拆成小事，然后一个一个搞定。

## 一、从"想到哪做到哪"到"有计划地做"

### 复习期末考试的两种方式

**不做规划的小明**：
> 随便拿起数学书开始看，看到哪算哪。结果数学看了 3 小时还没复习完，英语完全没碰。

**做规划的小红**：
> 1. 列出 5 门考试科目，评估每科的难度和自己掌握的程度
> 2. 分配时间：数学最弱，每天 2 小时；英语还行，每天 1 小时
> 3. 制定 7 天计划：前 3 天系统复习，中间 2 天刷题，最后 2 天总复习
> 4. 过程中发现某章比想象中难，及时调整

AI Agent 的**规划能力**，就和小红做复习计划一模一样。

### 为什么简单任务不需要规划？

你问 Agent"今天几号？"——它一步就能回答，不需要计划。

但如果你问 Agent"帮我写一份关于 AI 对教育影响的研究报告，要有最新数据、优缺点分析、专家观点"——这就不是一步能完成的了。Agent 需要：

1. 搜索 AI 在教育领域的应用现状
2. 搜索相关的统计数据
3. 搜索支持和反对 AI 教育的专家观点
4. 整理所有信息
5. 按照"背景 - 数据 - 观点 - 结论"的结构写成报告

Agent 不能一步完成，它得**先想好步骤，再一步步做**。

### 作业清单类比：三种规划方式

就像你写作业有不同的策略，Agent 规划也有三种：

| 方式 | 类比 | 适合什么 |
|------|------|----------|
| **先计划再执行**（Plan-and-Execute） | 先列出所有作业，然后一门一门写 | 步骤明确、顺序重要的任务 |
| **批量执行**（ReWOO） | 同时打开数学书、英语书、物理书，看哪科顺手先做哪科 | 步骤之间没有依赖的任务 |
| **多路径探索**（Tree of Thoughts） | 选志愿时列出几个方向，分别评估优缺点，选最好的 | 有多种可能路线的任务 |

### 班长分任务类比

班长要组织班级活动时怎么做的？

1. **先计划**：列出所有要做的事——定场地、买零食、排节目、发通知
2. **分配任务**：张三负责采购、李四负责场地、王五负责节目
3. **检查进度**：逐一确认每个任务完成没有
4. **遇到意外**：采购预算不够？调整方案，少买零食多买水果

Agent 的规划器也是这样工作的。

### 计划遇到意外怎么办？

完美计划也会翻车。比如你计划"今晚复习数学第三章"，但发现第三章比想象的难得多。

这时候你不能死守原计划。Agent 也一样：

1. **发现问题**：花了 30 分钟还没搞定这章
2. **分析原因**：这章的基础概念之前没学好
3. **调整计划**：今天先回头看第二章的基础知识，明天再攻第三章

这种"发现错了就改"的能力，叫**自我反思（Reflexion）**。

## 🤔 活动思考

### 活动一：规划大赛——筹办班级聚会（25 分钟）

**场景**：你是班长，要在下周为全班办期末庆祝会。

**任务**（3-4 人一组）：用"先计划再执行"方式制定完整计划。

计划必须包含：
- 至少 5 个步骤
- 每个步骤的依赖关系（哪步必须先做完）
- 每步可能遇到的意外和应对方案

**模板**：
```
目标：班级期末庆祝会

第1步: _____________  依赖: 无
  风险: _____________  应对: _____________

第2步: _____________  依赖: 第1步
  风险: _____________  应对: _____________

...
```

### 活动二：多路径探索大挑战（15 分钟）

**场景**：转学生想加入社团，有三个选择：篮球队、编程社、辩论队。

**任务**：
1. 各组分别列出加入某个社团的好处和坏处
2. 汇总后投票选出最佳建议

```

最终建议: ______  理由: ______
```

## 🔬 动手实验：任务规划器

```python
# 任务规划器：把大任务拆成小步骤

class TaskPlanner:
    def __init__(self, name):
        self.name = name
        self.plans = {}
        self.results = {}

    def make_plan(self, task):
        """制定任务计划"""
        print(f"\n正在为任务制定计划: {task}")

        if "旅行" in task:
            plan = [
                {"step": 1, "action": "确定目的地和时间", "depends_on": []},
                {"step": 2, "action": "查看天气预报", "depends_on": [1]},
                {"step": 3, "action": "查询交通方式", "depends_on": [1]},
                {"step": 4, "action": "搜索酒店", "depends_on": [3]},
                {"step": 5, "action": "规划每日行程", "depends_on": [4]},
                {"step": 6, "action": "整理预算", "depends_on": [3, 4]},
            ]
        elif "考试" in task or "复习" in task:
            plan = [
                {"step": 1, "action": "列出所有科目", "depends_on": []},
                {"step": 2, "action": "评估每科掌握程度", "depends_on": [1]},
                {"step": 3, "action": "按优先级排序", "depends_on": [2]},
                {"step": 4, "action": "制定每日复习计划", "depends_on": [3]},
                {"step": 5, "action": "收集复习资料", "depends_on": [4]},
            ]
        else:
            plan = [
                {"step": 1, "action": "理解任务要求", "depends_on": []},
                {"step": 2, "action": "搜索相关信息", "depends_on": [1]},
                {"step": 3, "action": "整理分析", "depends_on": [2]},
                {"step": 4, "action": "输出结果", "depends_on": [3]},
            ]

        self.plans[task] = plan
        print(f"\n{self.name}的计划:")
        for step in plan:
            deps = f"(需要先完成: {step['depends_on']})" if step['depends_on'] else "(无前置依赖)"
            print(f"  步骤{step['step']}: {step['action']} {deps}")
        return plan

    def execute_plan(self, task):
        """执行计划"""
        plan = self.plans.get(task)
        if not plan:
            print("没有找到计划，请先用 make_plan()")
            return

        print(f"\n开始执行计划...")
        completed = set()

        for step in plan:
            deps = set(step["depends_on"])
            if not deps.issubset(completed):
                print(f"  步骤{step['step']}等待: 依赖步骤{deps - completed}未完成")
                continue
            print(f"  步骤{step['step']}: {step['action']}... 完成!")
            self.results[step['step']] = f"{step['action']} 的结果"
            completed.add(step['step'])

        print(f"\n计划执行完成！完成{len(completed)}个步骤")


# 来试试！
planner = TaskPlanner("规划大师")
planner.make_plan("我想去杭州周末旅行")
planner.execute_plan("我想去杭州周末旅行")
planner.make_plan("下周要期末考试了，帮我制定复习计划")
planner.execute_plan("下周要期末考试了，帮我制定复习计划")
```

### 思考题

1. 看两个任务的计划。哪些步骤可以同时做（并行）？
2. 如果某个步骤失败了（比如搜不到酒店），规划器该怎么处理？
3. 你觉得是先计划再执行好，还是走一步看一步好？什么场景下哪种更好？
4. **挑战题**：实现"回滚"机制——当发现某步结果有问题时，退回上一步重新来。

## 💡 本章彩蛋

**你知道吗？** 2023 年有一篇很酷的论文叫《Tree of Thoughts》，它让 AI 像下棋一样思考——每走一步都想好几个可能性，评估每种可能，然后选最好的往下走。这就像你在《原神》里打深渊，先存档、试一种配队、不满意就读档换另一种配队。

还有一个有趣的概念叫 **Reflexion（反思）**——它让 Agent 在犯错后不是直接放弃，而是跟自己说"等一下，我刚才哪里做错了？下次应该怎么改？"这就像你考试后对答案，找出错题然后订正。

**回忆一下**：规划 = 拆任务 + 定顺序 + 执行 + 遇到问题就调整。下一章我们聊 Agent 的"记忆系统"——它怎么记住你！

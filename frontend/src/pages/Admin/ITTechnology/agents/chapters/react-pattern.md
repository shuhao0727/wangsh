# ReAct 模式：先想后做，边做边想

你有没有想过，为啥考试时有些人做题又快又准，有些人却手忙脚乱？

**小明**拿到卷子，拿起笔就写，写到一半发现理解错了题意，擦掉重写，又错了……最后时间不够。

**小红**先读题，想清楚"这题在考什么"，确定解题思路，写一步检查一步，发现不对立刻调整。

小红的做法就是 **ReAct 模式**的核心思想！

## 一、什么是 ReAct？

**ReAct** = **Rea**soning（推理）+ **Act**ing（行动）

翻译成人话就是：**每做一步之前，先想清楚为什么做、怎么做；做完之后看看结果，再想下一步。**

### 考试类比

```
你的考试流程（其实就是 ReAct）:
  第1步 思考: "这道题考三角函数的哪个公式？"
  第2步 行动: 写出公式，代入数字
  第3步 观察: 计算结果，检查是否合理
  第4步 思考: "结果看起来对，做下一题"
  第5步 行动: 开始解下一道题
  ...重复直到交卷
```

### 王者荣耀类比

打王者时，你也在用 ReAct：

- 看到敌方打野出现在小地图 → **观察**
- "他可能来抓我了，我先往塔下撤" → **思考**
- 按方向键往后退 → **行动**
- 看到他果然来了，但已经抓不到我 → **观察结果**
- "安全了，回去继续补兵" → **继续思考**

如果你不观察不思考，直接冲上去打——那就是"送人头"。

### ReAct 的三步循环

```
思考 → 行动 → 观察 → 思考 → 行动 → 观察 → ...
```

1. **思考（Thought）**：我现在知道什么？还需要知道什么？下一步做什么？
2. **行动（Action）**：真的去做！比如搜索、计算、发消息
3. **观察（Observation）**：看看结果怎样？得到了什么有用的信息？

然后回到第 1 步，继续。

### 三种 AI 工作方式对比

| 方式 | 会想吗？ | 会做吗？ | 适合什么？ | 容易出错吗？ |
|------|:---:|:---:|------|:---:|
| **纯推理** | 会 | 不会 | 数学证明题 | 可能想错，没法验证 |
| **纯行动** | 不会 | 会 | 简单搜索 | 像无头苍蝇乱撞 |
| **ReAct** | 会 | 会 | 复杂任务 | 错了能及时发现改正 |

### 一个真实的例子

假设你问 Agent："《三体》英文版是谁翻译的？他在哪工作？"

**纯推理（瞎猜）**：
> "可能是刘慈欣自己翻译的吧？"

**纯行动（瞎搜）**：
> 搜"翻译"→搜"三体"→搜"三体翻译"→搜了 8 次才找到

**ReAct（聪明地搜）**：
```
思考1: 这是两个问题。先查译者，再查他在哪工作。
行动1: 搜索"《三体》英文版 译者"
观察1: 译者刘宇昆（Ken Liu）

思考2: 译者是刘宇昆。现在查他的工作。
行动2: 搜索"刘宇昆 Ken Liu 职业"
观察2: 他是美籍华裔作家，毕业于哈佛大学

思考3: 信息齐全，可以回答了。
最终答案: 《三体》英文版译者是刘宇昆（Ken Liu），美籍华裔作家，哈佛大学毕业。
```

只用了 2 次搜索！因为每次搜索都基于上一次的结果，步步逼近。

## 🎮 类比理解

### 玩解谜游戏就是 ReAct

```
你玩密室逃脱的时候：

"我在第三个房间" ← 观察
"这个房间有个密码锁，密码应该藏在某处" ← 思考
"去翻抽屉和书柜找线索" ← 行动
"找到了！纸条上写着1984" ← 观察
"输入密码试试" ← 行动
"锁开了！" ← 观察结果
```

### Minecraft 中的 ReAct

```
你在 Minecraft 里想造一个房子：

思考: "我需要木头、石头、玻璃"
行动: 去砍树、挖矿、烧沙子
观察: "木头够了，石头还差一点"
思考: "先去挖更多石头"
行动: 继续挖矿
观察: "材料齐了"
行动: 开始建造
```

没有 ReAct 会怎样？你冲到空地上就开始乱放方块——最后造出一个四不像。

### 三种模式对比（一张图看懂）

```
纯推理（只在自己脑子里想）:
  想 → 想 → 想 → 写答案 
  问题：可能想错了也不知道

纯行动（只管做不看路）:
  做 → 做 → 做 → 做
  问题：方向可能一开始就错了

ReAct（想一步、做一步、看一眼）:
  想 → 做 → 看 → 想 → 做 → 看 → 想 → 完成 ✓
```

## 🔬 动手实验：ReAct 侦探

```python
# ReAct侦探：先想、再做、看了结果再继续

class ReActDetective:
    def __init__(self, name):
        self.name = name
        self.memory = []  # 侦探的笔记本
        self.step = 0

    def think(self, question, knowledge):
        """思考下一步该做什么"""
        self.step += 1
        print(f"\n[思考 {self.step}] {self.name}在思考...")

        if self.step == 1:
            if "电影" in question:
                return "search_movie"
            elif "城市" in question:
                return "search_place"
            elif "食物" in question or "吃" in question:
                return "search_food"
            else:
                return "search_web"
        else:
            if "还需要更多信息" in str(knowledge):
                return "search_web"
            else:
                return "give_answer"

    def act(self, action, question):
        """执行搜索动作"""
        print(f"[行动 {self.step}] {self.name}执行: {action}")
        database = {
            "search_movie": "电影《流浪地球》票房46亿，导演郭帆，主演吴京。",
            "search_place": "上海是中国最大城市，人口约2500万。外滩、东方明珠等景点。",
            "search_food": "火锅起源于重庆和四川，麻辣鲜香。毛肚、黄喉最受欢迎。",
            "search_web": "需要更多信息才能准确回答。",
        }
        return database.get(action, "未找到信息")

    def observe(self, result):
        """观察搜索结果"""
        print(f"[观察 {self.step}] {self.name}发现: {result[:60]}...")
        return result

    def solve(self, question):
        """ReAct主循环"""
        print(f"\n{'='*50}")
        print(f"问题: {question}")
        print(f"{'='*50}")

        knowledge = []
        max_steps = 3

        while self.step < max_steps:
            action = self.think(question, knowledge)
            if action == "give_answer":
                break
            result = self.act(action, question)
            observation = self.observe(result)
            knowledge.append(observation)
            self.memory.append({
                "step": self.step,
                "action": action,
                "result": result
            })

        print(f"\n[最终回答] {self.name}:")
        if knowledge:
            for i, k in enumerate(knowledge, 1):
                print(f"  {i}. {k}")
        else:
            print("  抱歉，没找到答案。")

        print(f"\n推理链:")
        for m in self.memory:
            print(f"  Step {m['step']}: {m['action']} → {m['result'][:40]}...")


# 试试你的ReAct侦探！
detective = ReActDetective("夏洛克·小A")
detective.solve("流浪地球这部电影是谁导演的？")
detective.solve("上海有什么好玩的地方？")
```

### 思考题

1. 观察"推理链"。你觉得哪一步最重要？为什么？
2. 如果第一次搜索结果不够准确，Agent 应该怎么做？
3. `max_steps` 设为 3 意味着什么？如果问题很复杂怎么办？
4. **挑战题**：给侦探添加"反问用户"的功能——不确定时问"你说的 XXX 是什么意思？"

## 💡 本章彩蛋

**ReAct 这个名字其实来自一篇 2022 年的论文**，作者在论文里证明：让 AI 边想边做，比只让它想或只让它做，准确率高出一大截。这个发现太重要了，以至于现在几乎所有的 AI Agent 都在用 ReAct 模式。

有趣的是，ReAct 的思想其实一点都不新鲜——你每天写作业、打游戏、做决定时，都在无意识地使用它。AI 科学家做的事情，不过是把人类最自然的思维方式"教"给了 AI。

**回忆一下**：ReAct = 思考 + 行动 + 观察，循环往复。下一章我们看看 Agent 怎么"做计划"——把大任务拆成小步骤！

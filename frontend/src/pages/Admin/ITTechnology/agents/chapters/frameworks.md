# 框架生态：AI 开发的"料理包"

## 你有没有想过...

你吃过自热火锅吗？打开包装，把料理包倒进去，加水，等 15 分钟——一锅火锅就好了。你不会觉得"这不是正经火锅"，因为料理包里有牛油锅底、有各种涮菜，味道还真不赖。

但如果让你从零开始做火锅呢？买牛骨熬汤底（6 小时）、手工做牛肉丸（2 小时）、切羊肉卷、调蘸料...你忙活一整天，可能还不如自热火锅好吃。

做 AI Agent 也是这个道理。**框架就像料理包——不用从零开始，但你要知道里面有什么、怎么搭配最好吃。** 没人会从零写 AI 的"大脑"（那需要烧几百万美元训练大模型），聪明的开发者都用现成的框架，像拼乐高一样搭出自己想要的 AI。

---

## 两大流派：编剧 vs 即兴演员

市面上的 AI 框架分成两大"门派"，就像两种不同的表演风格：

**编排派 = 舞台剧演员（有剧本的）**
- 代表：LangChain、LangGraph
- 特点：提前写好"剧本"（流程图），AI 必须按剧本一步一步来，不能自由发挥
- 优点：行为可控，出问题一眼就知道"第几幕台词说错了"
- 缺点：如果观众提的问题不在剧本里，NPC 就卡住了
- 就像原神里的剧情对话——选项是事先写好的，你不能说"我不想选这三个，我要自己编一句"

**对话派 = 即兴喜剧演员**
- 代表：AutoGen、CrewAI
- 特点：AI 们自由对话，"聊着聊着就把活干了"
- 优点：灵活，给你惊喜
- 缺点：有时候聊着聊着就跑偏了，像群聊里的话题突然从"数学作业"歪到"晚上吃什么"

两派没有谁好谁坏——就像舞台剧和脱口秀，场景不同，选择不同。银行转账系统肯定用编排派（精确可控），聊天陪伴 AI 就更适合对话派（自然灵活）。

## 🎮 类比理解

**CrewAI 就像你在王者荣耀里开黑**：你是指挥（项目经理），中路法师是"研究员 Agent"（负责搜集信息），打野是"执行 Agent"（负责实际操作），辅助是"检查 Agent"（负责把关）。你给每个人分配任务，大家各司其职。有人搞砸了？调度一下，让其他人补位。

**LangGraph 就像 Minecraft 里的红石电路**：每个红石元件（节点）做一件事，红石粉（边）连接它们，信号（数据）在电路上流动。你可以设计"如果前面的门开了，后面的活塞才启动"这种条件逻辑。精确、可控、出错好找。

**AutoGen 就像原神的多人联机**：三个玩家进了同一个世界，没有任何预设的"任务流程"。一个说"我们去打这个 boss 吧"，另一个说"等等我先去采点药"，第三个说"我觉得我们现在伤害不够"。自由讨论中自然形成分工——聊着聊着就把 boss 打了。

## 货架上的"料理包"大盘点

### LangChain：瑞士军刀
资格最老，功能最全。核心思想是把 AI 开发拆成"积木"：提示词模板是一块，模型调用是一块，结果解析是另一块。你把这些积木串成**链（Chain）**，就像流水线——第一个同学出题，第二个同学做题，第三个同学批改。

LangChain 的升级版叫 **LangGraph**，把"一条线"升级成了"一张图"。可以画成分支结构："如果结果正确，就输出；如果不正确，就重试；如果重试超过 3 次，就呼叫人工帮助。"这就像游戏里的分支剧情。

### CrewAI：AI 团队指挥官
设计思路超简单：**把 AI 当你的队友。** 你只需要做三件事：
1. 定义一个 Agent（"你是搜索专家"）
2. 分配一个 Task（"去查出今天北京的天气"）
3. 把 Agent 和 Task 绑在一起，开工！

就像一个班主任分配大扫除：张三擦窗户、李四扫地、王五倒垃圾——各自有各自的任务。

### AutoGen（微软出品）：AI 聊天室
核心理念：**让 AI 们自己聊天解决问额。** 把一群 AI 拉进"群聊"，给个话题，它们自己讨论分工。最酷的是它能自己写代码并运行——一个 AI 写 Python 脚本，另一个运行看结果，出错了还自己改。

### LlamaIndex：图书管理员
如果你想让 AI 读文档、查资料、翻数据库，这是你的首选。你把一堆资料扔给它，它自动建索引、分类。你问"这本书里提过牛顿第二定律吗"，它一秒翻完整个图书馆。

### Dify：零代码搭建平台
你还不会写代码？没关系。Dify 是**可视化拖拽**平台，跟搭积木一模一样——拖一个"聊天框"模块、拖一个"知识库"模块、连上线、点发布——你的 AI 应用就上线了！

## 怎么选？一张表搞定

| 你想做什么 | 推荐框架 | 为什么 |
|-----------|---------|--------|
| 刚入门，想做个好玩的 | CrewAI | 概念少，上手快，像玩角色扮演游戏 |
| 需要精确控制每一步 | LangGraph | 画流程图式开发，指哪打哪 |
| 让 AI 读大量文档 | LlamaIndex | 自带"图书馆管理"超能力 |
| 让 AI 自由讨论解决问题 | AutoGen | AI 群聊，很有趣 |
| 深度使用 OpenAI | Agents SDK | 官方出品，跟模型天生一对 |
| 还不会写代码 | Dify | 拖拖拽拽就能做出 AI 应用 |

## 动手试一试：组建 AI 小队

```python
# 模拟 CrewAI：组建一个"学习助手小队"
import random

class Agent:
    def __init__(self, name, role, skill):
        self.name = name
        self.role = role
        self.skill = skill  # 1-10

    def work(self, task):
        success = random.random() < (self.skill / 10)
        emoji = "✅" if success else "❌"
        print(f"  [{self.role}] {self.name}：{task} {emoji}")
        return success

# 组建团队
researcher = Agent("小搜", "搜索专家", 8)
planner   = Agent("小计", "计划达人", 9)
tutor     = Agent("小讲", "辅导老师", 7)
checker   = Agent("小检", "质检员", 8)

print("🎓 AI 学习助手小队集合完毕！\n")

# 分配任务
tasks = [
    (researcher, "搜索近三年高考数学真题"),
    (planner, "制定本周数学复习计划"),
    (tutor, "讲解导数概念"),
    (checker, "检查讲解是否有误"),
]

for agent, task in tasks:
    agent.work(task)

print("\n💡 这就是 CrewAI 的本质 —— 角色分配 + 任务调度 + 团队协作！")
```

## 💡 本章彩蛋

你知道吗？大多数流行的 AI 框架都只有几岁。LangChain 诞生于 2022 年底，CrewAI 在 2023 年底，AutoGen 和 Agents SDK 都是 2024 年的新东西。这意味着你现在学的东西，跟世界上最顶尖的 AI 开发者用的是完全一样的技术——没有任何"代差"！

而且这些框架全都是开源的，代码在 GitHub 上随便看。你可以在 LangChain 的代码里搜 `# TODO`，找到那些开发者还没来得及实现的功能，然后...也许你就是下一个给 LangChain 贡献代码的人呢？

**记住：框架是工具，不是魔法。好厨师用同样的料理包能做出更好吃的菜，关键在于你理解里面有什么、知道怎么调。**

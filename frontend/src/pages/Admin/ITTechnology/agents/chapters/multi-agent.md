# 多智能体协作：AI 也"开黑"

你有没有想过，为什么王者荣耀一个人再强也赢不了？因为这是五个人的游戏——需要有人抗伤害、有人打输出、有人保护后排、有人抓人、有人推塔。

AI 也是一样！单个 AI Agent 就像一个"全能选手"——又要搜索、又要计算、又要写作、又要检查，忙得焦头烂额还容易出错。所以工程师发明了**多智能体系统**，让一群各有所长的 AI 像战队一样协作。

## 一、为什么"独狼"不够用？

### 想想你做小组作业的时候

老师让你一个人完成"资料收集 + PPT 制作 + 演讲汇报"。你发现：
- 一个人查资料效率低，容易漏掉关键信息
- 做到后面忘了前面，逻辑容易乱
- 没人帮你检查，小错误自己很难发现

单个 AI Agent 的困境和你一模一样：
- **"脑子"不够用**：AI 的记忆有上限，信息太多就会"忘记"
- **技能太杂**：一个 Agent 什么都要会，结果样样通样样松
- **没有"后悔药"**：走错一步没人提醒，错误像多米诺骨牌一样传递

### 多智能体 = 一支 AI 战队

核心思想很简单：**把大任务拆成小任务，每个 AI 负责自己最擅长的部分**。

就像王者战队：

| 游戏角色 | 职责 | AI 类比 |
|---------|------|---------|
| 队长/指挥 | 制定策略、分配任务 | **规划者**：把需求拆成具体步骤 |
| 各路线选手 | 对线、打团、推塔 | **执行者**：接收指令，动手干活 |
| 教练/分析师 | 赛后复盘、指出问题 | **评审者**：检查结果，挑毛病 |
| 团队沟通 | 报点、发信号 | **协调者**：管理消息，避免"撞车" |

---

### AI 之间怎么"说话"？

你和队友开黑时怎么沟通的？AI 也一样有三种方式：

**方式一：全队语音（广播模式）**

所有人都能听到所有人的话。适合 5 人以下的小队。人一多就吵成一团——AI Agent 数量多了也会"炸麦"。

**方式二：私聊/小队语音（定向通信）**

只跟需要配合的队友说话。ADC 和辅助开私聊，不影响上单。高效安静，但需要有人知道"谁该跟谁说话"。

**方式三：指挥链（层级通信）**

像军队一样层层传达：

```
总指挥（规划者）
  ├── 小队长A
  │    ├── 执行兵A1
  │    └── 执行兵A2
  └── 小队长B
       ├── 执行兵B1
       └── 审查兵B
```

命令从上往下传，结果从下往上报。适合超大任务，但传话环节多了可能"走样"。

### 意见不合怎么办？——AI 辩论赛

两个 AI 对同一问题给出不同答案，怎么办？这时候需要**辩论机制**。

就像课堂辩论：
1. 正方（Agent A）和反方（Agent B）各自提出方案
2. 双方互相质疑、各自改进，在限定的回合数内"唇枪舌剑"
3. 裁判（一个更公正的 AI，或人类老师）根据辩论记录给出最终裁决

除了辩论，还可以**投票**——三个 AI 各自独立判断，少数服从多数。就像你们班选班长，得票最多的当选。

### 任务怎么分配？——像老师排值日

老师安排值日分工时：
- **固定分配**：像固定的值日表——周一张三、周二李四。简单，但不灵活。
- **自愿认领**：像"这道题谁来答？"——觉得自己会的举手，老师选一个。
- **智能分配**：由"班长 AI"根据任务描述，判断谁最合适。

### 主流多智能体框架

| 框架 | 一句话描述 | 适合谁 |
|------|-----------|--------|
| **CrewAI** | 像组建团队：定义角色、分配任务、开工 | 初学者 |
| **AutoGen（微软）** | AI 之间自由聊天，聊着聊着把活干完 | 需要灵活对话 |
| **LangGraph** | 画一张"流程图"，AI 按图纸走 | 需要精确控制流程 |
| **MetaGPT** | 模拟软件公司：PM、程序员、测试各司其职 | 软件开发任务 |

## 🎮 类比理解

### 王者荣耀团战 vs 多 Agent 协作

```
王者团战：
  辅助: 做视野，发现敌方位置 → 发信号
  打野: "对面打野在下路，我抓上路" → 行动
  中单: "我清完线就来支援" → 协调
  射手: "等我装备成型再开团" → 反馈
  上单: "我先抗伤害，你们跟上" → 配合

多Agent协作：
  搜索Agent: 找到相关数据 → 报告结果
  分析Agent: "数据还不够，需要更详细的" → 反馈
  写作Agent: "我先写初稿，你们补充" → 协调
  评审Agent: "第三段逻辑有问题，需要改" → 检查
  规划Agent: "按这个顺序来，先搜后写再审" → 统筹
```

### 班级小组合作类比

```
老师布置大作业："制作一份关于环保的展示"

不做分工（一个人干）:
  你一个人查资料、做PPT、写演讲稿、练习汇报
  → 累死，PPT 很简陋，演讲时忘词

做好分工（多人协作）:
  组长: 制定计划，分配任务，检查进度
  资料员: 搜索环保数据、找图片、整理素材
  设计员: 制作 PPT，排版美化
  演讲员: 写演讲稿，练习，最终汇报
  质检员: 检查 PPT 有没有错别字，数据准不准
```

## 🤔 活动思考

### 活动一："无声协作"挑战（25 分钟）

**场景**：班级科技节项目——设计"校园智能导航"方案。四个子任务：
1. 数据调研（测量校园各地点距离）
2. UI 设计（画界面草图）
3. 算法设计（找最短路径）
4. 文案撰写（写项目说明书）

**规则**：
- **第 1 轮**：你一个人完成全部，计时 15 分钟
- **第 2 轮**：4 人团队，每人负责一项，但**不能说话**（模拟广播被关），只能写纸条传递
- **第 3 轮**：4 人团队，可自由交流，但需要一个"组长"统筹

**讨论**：
1. 第 2 轮和第 1 轮比，效率提升了还是下降了？
2. 第 3 轮加入"组长"后发生了什么变化？
3. 这个实验说明了多智能体系统的哪些关键点？

### 活动二：AI 辩论裁判（15 分钟）

**场景**：三个"AI 辩手"对"高中生每天应该玩多长时间手机？"给出不同答案：
- AI-A：不超过 30 分钟
- AI-B：1-2 小时没问题，关键是自律
- AI-C：看情况，考试周少，假期多

**任务**：
1. 你扮演"裁判 Agent"，为每个方案打分（1-5 分）
2. 如果让这三个 AI 互相辩论 3 轮，你预测结果会怎样？
3. 设计"投票规则"：如果有 5 个 AI 投票，怎么保证公平？

## 🔬 动手实验：多智能体协作模拟

```python
# 小型多智能体模拟：三个AI协作完成任务
import random

class PlannerAgent:
    """规划者：把复杂问题拆成步骤"""
    def plan(self, question):
        print(f"[规划者] 收到问题：{question}")
        steps = [
            "步骤1：提取题目信息",
            "步骤2：选择解题方法",
            "步骤3：计算求解",
            "步骤4：验算结果"
        ]
        print(f"[规划者] 拆解为 {len(steps)} 个步骤")
        return steps

class ExecutorAgent:
    """执行者：按步骤干活"""
    def __init__(self, name):
        self.name = name

    def execute(self, step, data):
        print(f"[{self.name}] 执行 {step}...")
        if random.random() < 0.85:  # 85% 成功率
            print(f"  {self.name} 成功!")
            return {"success": True, "result": f"{step} 完成"}
        else:
            print(f"  {self.name} 出错了！")
            return {"success": False, "result": None}

class ReviewerAgent:
    """评审者：检查执行结果"""
    def review(self, results):
        print(f"[评审者] 正在检查 {len(results)} 个结果...")
        passed = sum(1 for r in results if r["success"])
        failed = len(results) - passed
        success_rate = passed / len(results) if results else 0
        print(f"[评审者] 通过率：{success_rate:.0%} ({passed}/{len(results)})")
        return success_rate >= 0.8  # 超过80%就算通过


# 跑起来！
print("=" * 50)
print("   多智能体协作模拟")
print("=" * 50)

planner = PlannerAgent()
math_guy = ExecutorAgent("数学达人")
logic_guy = ExecutorAgent("逻辑高手")
reviewer = ReviewerAgent()

steps = planner.plan("计算圆的面积，半径=5")

results = []
for i, step in enumerate(steps):
    worker = math_guy if i % 2 == 0 else logic_guy
    result = worker.execute(step, f"中间结果_{i+1}")
    results.append(result)

passed = reviewer.review(results)
if not passed:
    print("\n--- 第一轮不通过，启动第二轮（重做失败的步骤）---")
    for i, r in enumerate(results):
        if not r["success"]:
            worker = math_guy if i % 2 == 0 else logic_guy
            results[i] = worker.execute(steps[i], f"返工_{i+1}")
    passed = reviewer.review(results)

if passed:
    print("\n任务完成！团队成功交付！")
else:
    print("\n多次尝试未达标，需要人工介入。")

print("\n思考：如果只有一个Agent，出错怎么办？")
print("多Agent的好处：有人检查、有人兜底！")
```

### 思考题

1. 如果把执行成功率从 85% 改成 60%，系统会发生什么？
2. 如果评审者不仅能发现问题，还能**给出修改建议**，代码怎么改？
3. 如果再加一个 `CoordinatorAgent`（协调者），它应该在哪个位置？

# 微调与对齐：从"会答题"到"真正懂你"——AI的家教课

## 🎯 读完本章你能...

理解LLM训练的后两个阶段——监督微调（SFT）和人类反馈对齐（RLHF），知道预训练完的模型只是"半成品"（它会续写但不会当助理），知道SFT是用"精品问答对"教模型行为规范，知道RLHF是用"人类偏好排序"让模型更安全更有用。了解国产模型（DeepSeek、Qwen、ChatGLM、文心一言）在对齐阶段做了哪些独特的努力。

## 📖 从一个故事开始

想象你有一个"过目不忘"的天才朋友。他读了整个图书馆的书——物理、化学、历史、哲学、编程、文学——什么都懂。但有一个问题：你跟他说"我失恋了，好难过"的时候，他会一本正经地告诉你"失恋是哺乳动物在社会性挫折后的常见心理应激反应，从进化心理学角度看……"而不是说"抱抱你，要不要一起喝杯奶茶？"

你急了："能不能像个正常人一样说话？"

天才朋友愣住了："什么是'像个正常人'？怎么定义'正常'？"

这就是预训练完的LLM的真实状态。它读遍了互联网，知识渊博到令人恐惧，但它**不知道自己是助理**。你跟它说"写一首诗"，它可能给你列出从《诗经》到现代诗的发展简史——因为它"学"到的文本模式里，提到"写诗"后面经常跟着学术分析。它没有"当助理"的直觉——它就是一个高效的"文本续写器"。

接下来的两个阶段——**监督微调**（SFT）和**人类反馈对齐**（RLHF）——就是要把这个"知识渊博但情商为负的续写机器"，调教成一个"懂你、帮你、不害你"的AI助手。

这就像你把一个天才学霸送去上一对一家教课——不是教他新知识（他的知识已经够了），而是教他"怎么跟人说话"、"怎么判断该说什么不该说什么"、"怎么说才让人觉得舒服"。

## 📖 原理讲解

### 三阶段训练全景回顾

在深入微调和对齐之前，先看一眼全貌。现代LLM的训练是标准的三步流水线：

| 阶段 | 做什么 | 数据量 | 像什么 |
|------|--------|--------|--------|
| 预训练 | 海量阅读，学会语言规律 | TB级别 | 从小学到高中十二年教育 |
| SFT | 做精品题，学会行为规范 | 几万到几十万条 | 考前做真题 |
| RLHF/DPO | 比较偏好，学会安全有用 | 几万到几十万条 | 私教一对一纠错 |

预训练（上一章）让模型成为"博学的人"。SFT让它成为"能用的人"。RLHF让它成为"好用的人"。

### SFT——做精品题，学"好答案长什么样"

SFT（Supervised Fine-Tuning，监督微调）的原理极其简单，但执行起来需要极高的耐心。

**做法**：找一群标注员（或资深专家），写几万条"问题-高质量回答"对。格式如下：

```
<用户>：怎么才能学好英语？
<助手>：学好英语需要三个维度的努力：
1. 大量输入——每天至少听、读30分钟英语材料（播客、新闻、小说）
2. 刻意输出——找机会开口说和写，哪怕一开始磕巴
3. 纠错反馈——找英语母语者或好老师帮你纠正，错误不改正会养成习惯
关键是坚持——每天一小时，一年后绝对不一样。
```

然后在这些"精品对"上继续训练模型——就像预训练时一样做"预测下一个词"，但数据不再是互联网上的随便什么文本，而是精心编排的对话。

SFT的数据不需要海量——几千到几万条就足以让模型的行为发生质变。关键不是**多**，而是**覆盖全面**和**质量极致**。

覆盖全面的意思是：各种类型的问题都要有——聊天、创作、代码、数学、情感支持、知识问答。如果只在编程数据上微调，模型会变成一个"只知道写代码的AI"，你跟它聊天它都给你输出Python。

质量极致的意思是：每一条回答都要反复打磨。有一个广受认可的发现是：

> 1000条精心打磨的高质量SFT数据，效果远胜10000条平均水平的SFT数据。

这不是"越多越好"的事情——是"越精越好"。就像你考前做真题：做10套真题但每套都认真对答案、总结错因，比做100套但不批不改有效得多。

国产模型在SFT上有独特的优势：因为主要面向中文用户，国内团队在"中文表达的地道性"上投入了大量精力。比如：
- **Qwen团队**对中文SFT数据做了极其细致的打磨，特别注重中文里含蓄表达、语境暗示的理解。
- **DeepSeek团队**收集了大量中文语境下的专业问答数据，覆盖理工、医疗、法律、金融等垂直领域。
- **文心一言**利用百度搜索的意图理解数据，在"用户模糊表达→精准理解"上做了专项优化。

### RLHF——老师打分，学会"高分答案"的套路

SFT之后的模型已经能当助理了，但它还有几个核心问题：

1. **会胡说**：它可能自信满满地输出错误信息——因为它不知道"什么是真的"。
2. **会冒犯**：它可能输出不安全、有偏见的内容——因为训练数据里就有这些。
3. **会绕弯**：它可能给出"正确但没人想看"的回答——又臭又长、不抓重点。

RLHF（Reinforcement Learning from Human Feedback，从人类反馈中强化学习）就是为了解决这些"最后一公里"的问题。它的流程分为三步：

**第一步：收集人类偏好数据**

对于同一个问题，让模型生成多个不同回答。人类标注员判断"哪个回答更好"。

```
问：怎么减肥？
答A：每天跑步一小时，控制饮食，坚持三个月。（好回答——具体、可操作）
答B：减肥就是少吃多动就行了。（差回答——太笼统、毫无帮助）
→ 标注员标记：A > B
```

几千到几万条这样的"偏好对"被收集起来。

**第二步：训练奖励模型（RM）**

用这些偏好对训练一个"打分器"——输入是（问题，回答），输出是一个分数，表示"人类会多喜欢这个回答"。

奖励模型本身也是一个神经网络（通常从SFT模型初始化），它学会的是：给好回答打高分，给差回答打低分。它不需要理解"为什么"——它只需要学会人类标注员的判断模式。

**第三步：用强化学习优化LLM**

训练方式转变为"对话回合制"：LLM对某个问题生成回答 → 奖励模型给他打分 → 用PPO（近端策略优化，一种强化学习算法）的规则，根据分数来调整LLM的参数，让它更倾向于生成高分回答。

这个过程反复进行几千到几万轮，模型逐步学会：什么样的回答"人类喜欢"。

RLHF的效果是显著的：经过RLHF的模型比只经过SFT的模型，在"回答质量"、"安全性"、"诚实度"等维度上都有明显提升。而且有一个有趣的现象：RLHF实际上让模型学会了一种"品质直觉"——它不需要被明确告知"不要说脏话"，但在奖励模型反馈几万次后，它自然而然地避免生成粗俗内容。

### DPO——更简洁的"直接偏好优化"

RLHF有一个工程上的痛点：需要额外训练一个"奖励模型"（第二步），这让整个流程又复杂又昂贵——你要训两个模型（奖励模型+要优化的LLM），而且要协调它们之间的训练。

2023年，斯坦福的研究团队提出了**DPO**（Direct Preference Optimization，直接偏好优化），核心思想是：**把人类的偏好直接作为LLM的损失函数，不需要单独的奖励模型**。

DPO的数学推导相当精妙，简单说就是：在偏好数据"回答A好于回答B"中，你直接让LLM"提高生成A的概率、降低生成B的概率"。这个优化目标可以用一个简洁的二元交叉熵损失表达，不需要强化学习那套复杂流程。

DPO出来后迅速流行，因为它更简洁、更稳定、更容易实现。很多开源模型（包括Qwen 2.5的后续版本、ChatGLM-4等）在RLHF阶段混合使用了DPO和传统RLHF，取得了很好的效果。

不过DPO也不是万能的：它在数学和编程等"有明确正误"的任务上效果好，但在"风格偏好""创意写作"等主观领域，传统的RLHF+奖励模型方案仍然有优势。

### 安全对齐——国产模型的"中国特色"挑战

对齐不只是"回答问题好一点"——它还涉及**安全对齐**（Safety Alignment），即确保模型不会输出有害、违法、违法违规的内容。

这对中国的大模型来说是一个特别复杂的挑战，因为中文语境下的安全需求是多层次的：

- **法律合规**：不输出违反中国法律法规的内容。
- **社会主义核心价值观对齐**：符合中国社会的主流价值取向。
- **文化敏感性**：理解中文语境中的禁忌、暗示和隐喻。
- **青少年保护**：面对可能的未成年用户时，输出适合的、安全的内容。

国内各团队的做法：

- **深度求索（DeepSeek）**在安全对齐上走了一条"技术攻关"的路线——提出了"无监督安全对齐"的方法，试图在不需要大量人工标注的情况下，让模型自己理解"什么是不安全的"。他们还开源了安全评估数据集，推动整个中文AI生态的安全标准建设。
- **阿里（通义千问）**利用阿里的电商和内容审核经验，构建了多层安全过滤机制——输入阶段、生成阶段、输出阶段各有一层安全检查。
- **百度（文心一言）**结合搜索能力，在模型可能生成敏感内容时主动检索权威信息来补充或纠正。
- **智谱（ChatGLM）**与清华合作，在学术层面对"AI价值观对齐"做了大量基础研究。

安全对齐不是一个"做完就完"的事情——它是一个持续的过程。新的攻击方式（越狱攻击、提示注入）不断出现，对齐技术也需要不断进化。这就像游戏里你和外挂开发者的"军备竞赛"——你刚封了一个漏洞，对方又找到三个新的。

### 国产模型的微调生态

预训练一个LLM需要几百万到上千万美元，大多数公司和个人玩不起。但**微调**一个已经预训练好的模型，只需几百到几千美元——这使得"在开源模型上微调"成为中国AI应用最活跃的生态。

以Qwen为例：阿里把它从0.5B到72B的模型全部开源，开发者下载72B的Qwen，用自己的专业数据（比如医疗问诊对话、法律案例问答、游戏客服回复）做SFT和DPO，就能得到"Qwen-医疗版"、"Qwen-法律版"、"Qwen-游戏客服版"。HuggingFace上已经有几千个基于Qwen的微调变体。

DeepSeek的定价策略更是直接催化了这个生态：它的API价格极低（输入仅0.27元/百万token），这意味着小团队不需要昂贵的硬件就能用上世界级的大模型。很多人用DeepSeek做"蒸馏式微调"——用DeepSeek先生成一批高质量训练数据，再用这些数据微调一个小得多的本地模型（如Qwen-7B），实现"花小钱办大事"。

## 🎮 类比理解

LLM的三阶段训练就像你准备《原神》深境螺旋12层：

- **预训练 = 氪金抽角色、练等级、刷圣遗物**：你花了几个月把角色池扩充到几十个，每个角色都练到了90级，圣遗物也刷了几百套。这是"硬实力"的积累——角色池深度决定了你能打的多远。
- **SFT = 找攻略视频看配队和手法**：你看了几十个高手的深渊攻略视频，知道了"胡桃+行秋+钟离+夜兰"的胡行钟夜组合、知道了"万叶双扩散"的手法。你模仿这些"精品示范"，学会了"正确操作长什么样"。
- **RLHF = 自己打，反复重开，每次打完分析"这层为什么没满星"**：你在一次次实战中，自己总结出"第三间上半留大招给下半能快5秒"、"12-3的雷音权限走位要往左边闪"这些"攻略里没写但实战中有用"的微操技巧。RLHF就是这种"从实战反馈中自我进化"的过程。

DPO就是：你不用反复打——你直接把大神满星的录屏和你自己翻车的录屏放在一起对比，然后告诉自己"学大神的手法，别学自己翻车的操作"。少了反复打的过程，直接比较最优和最差，效率更高。

## 💡 本章彩蛋

RLHF这个概念其实来自于DeepMind在2017年的一篇论文。但真正把它"用出花"的是OpenAI——2022年发表的InstructGPT论文详细描述了怎么用RLHF把GPT-3变成"能听从指令的助手"，这也是ChatGPT背后的关键技术。

但更让人惊讶的是：Anthropic（Claude的开发公司）在RLHF基础上发展出了一套叫**宪法AI**（Constitutional AI）的方法。它的核心思想是：与其让人一条一条标注"这个回答好、那个回答差"，不如写一套"AI宪法"——比如"不要输出有害内容""要诚实表示不确定""要尊重用户隐私"——让AI自己按照这套宪法来批评和修订自己的回答。这被称为**RLAIF**（从AI反馈中强化学习，而非从人类反馈）。

2024年，中国深度求索（DeepSeek）更进一步——他们在DeepSeek-V3中探索了"纯SFT+DPO"路线，几乎跳过了传统RLHF步骤，仍然取得了顶尖的对齐效果。这对RLHF是否需要"强化学习"这个复杂框架提出了有趣的质疑。

另外：你可能注意到，ChatGPT有时会拒绝回答一些"看起来挺正常"的问题。这是RLHF的结果——在人类标注员的数据中，某些话题被系统性地标记为"不安全"，于是模型学会了在这些话题上"三缄其口"。RLHF让模型更安全，但也让它更"谨慎"——这是一种有意为之的权衡。OpenAI的CEO Sam Altman自己都承认："我们可能把模型调得过于谨慎了。"

最后一个小故事：Anthropic的Claude之所以"特别有礼貌"和"特别谨慎"，是因为它在RLHF阶段被灌输了大量的"礼貌性偏好"——人类标注员被明确告知"优先选择礼貌的、谦逊的、顾及用户感受的回答"。所以Claude总是说"你提出了一个很好的问题""让我确认一下我理解了你的意思"——这不是Claude的性格，是人类标注员的集体性格投射在它身上。

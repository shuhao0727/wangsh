# LLM预训练：吃下整个互联网——从海量文本到超级大脑

## 🎯 读完本章你能...

理解LLM（大语言模型）预训练的完整流程——从收数据（TB级文本清洗）到分词（把汉字切成一串ID）到训练（让模型预测下一个词），知道扩增法则（Scaling Law）的核心发现——模型越大、数据越多、算力越强，效果就越好（简单粗暴但烧钱），了解国内外主流预训练模型的格局，特别是国产模型（DeepSeek、通义千问、文心一言、ChatGLM）怎么在这个赛道里杀出一条路。

## 📖 从一个故事开始

你有没有过这样的体验：你读了很多很多书——教科书、网络小说、新闻评论、游戏攻略、菜谱、代码文档、论文——虽然没有"刻意记"任何一本，但某天你跟朋友聊天时突然引经据典，自己都惊讶"我什么时候知道了这个？"

这就是**海量阅读**的力量。你不需要背诵任何一本书，但读了够多之后，你对"什么是合理的句子""什么论点有说服力""怎么表达更高效"产生了深入骨髓的直觉。

大语言模型的预训练就是这件事：让模型"读"互联网上几乎所有能找到的文本——维基百科、书籍、论坛帖子、新闻、论文、代码仓库、技术博客……读完后，它不记得任何具体的文本，但它学到了语言本身的深层规律。

这个过程叫**预训练**（Pre-training），是LLM训练的第一阶段，也是最烧钱、最漫长、最"大力出奇迹"的阶段。如果说整个LLM训练像你准备高考，那预训练就是你从小学到高中的全部阅读和知识积累——没有这十几年的积累，什么"考前刷题""名师辅导"都是空中楼阁。

## 📖 原理讲解

### 数据：预训练的"粮食"

预训练的第一步和最重要的一步，是收集和清洗数据。有一句话在LLM圈子里被反复提及：**"垃圾进，垃圾出"（Garbage In, Garbage Out）**。数据质量直接决定了模型的上限。

现代LLM的预训练数据规模已经大到令人咋舌：

| 模型 | 训练数据量 | 大概相当于 |
|------|-----------|----------|
| GPT-1 (2018) | ~5GB | 约7000本书 |
| GPT-2 (2019) | ~40GB | 约56000本书 |
| GPT-3 (2020) | ~570GB | 约80万本书 |
| LLaMA 2 (2023) | ~2TB | 约280万本书 |
| DeepSeek-V3 (2024) | ~14.8TB | 约2000万本书 |

14.8TB纯文本是什么概念？如果一个人一辈子每天读8小时、每分钟读300字，他这辈子大概能读完1-2GB的纯文本。DeepSeek-V3的预训练数据量大约是一个人的一万辈子。

但光是"多"不够，还得"好"。预训练数据的处理是一个极其精细的工程：

- **去重**：同样的文章出现在100个网站上，只保留一份。否则模型会对重复内容产生偏见。
- **质量过滤**：去掉明显是垃圾的内容（乱码、无意义重复、机器生成的灌水文章）。
- **毒性过滤**：去掉仇恨言论、极端暴力内容、个人信息泄露。
- **语言配比**：决定中文占多少、英文占多少、代码占多少——不同配比训练出来的模型能力完全不同。
- **时效性**：太老的数据（比如2005年的技术文章）和太新的数据（T+1日的实时内容）各有利弊，需要平衡。

这项工作的枯燥程度超乎想象——就像你在游戏里做"全图鉴收集"：要把几千个物品逐一收集、分类、去重、整理，还不能漏掉重要的稀有物品。

### 分词——把汉字变成模型能"吃"的ID

模型不能直接理解汉字或英文单词——它只能理解数字。**分词器**（Tokenizer）就是把文本转换成数字序列的工具。

分词的粒度选择是一个经典的工程权衡：

**按字分**：每个汉字一个ID。"我爱中国"→ [我, 爱, 中, 国] → [234, 891, 45, 120]。优点是词表小、没有OOV（词表外）的问题；缺点是序列太长、效率低。

**按词分**：每个词一个ID。"我爱中国"→ [我, 爱, 中国] → [234, 891, 1023]。优点是序列短、语义完整；缺点是词表巨大（中文有几万个常用词），而且遇到新词就废了。

**BPE（字节对编码）**：现代LLM最常用的折中方案。从字符级别开始，统计文本中"哪两个符号经常在一起出现"，把最高频的组合合并成一个新符号。反复执行这个过程，直到词表达到预设大小。

BPE的结果很神奇：高频词"中国"保留为完整的一个token，低频词"秦始皇陵兵马俑"可能被切成"秦始皇"+"陵"+"兵马俑"三个token。常用词不被打散（效率高），罕见词被切成碎片（不会出现词表外的词）。

你可以在OpenAI的在线工具上玩一下分词器：输入一段中文，看它被切成什么样的token序列。你会发现"我喜欢DeepSeek"被切成的不全是单个汉字——模型实际上"看"到的世界是一串token ID。

### 训练目标：史上最简单的任务——"猜下一个词"

数据有了，分词完了，模型要做什么来学习？

预训练的任务出奇地简单：**给模型看一段文字的前半部分，让它预测下一个词是什么。**

> 输入："地球是太阳系中唯一已知存在"
> 模型的任务：预测下一个token

这个任务叫**语言建模**（Language Modeling），也叫**下一个Token预测**（Next Token Prediction）。它简单到令人困惑：就这？就靠猜下一个词，能学会理解和推理？

答案是：能。而且是惊人的能。

原因在于：**"猜下一个词"这个看似简单的任务，隐含了对世界的深度理解**。

要准确预测"地球是太阳系中唯一已知存在___"的下一个词，你至少要理解：
- 地球是天体（领域知识）
- 太阳系包含多个天体（天文知识）
- "唯一已知存在"暗示后面要接一个关于地球独特性的描述（语法和语义理解）
- 在人类现有的天文学知识框架下，地球的独特性是"存在生命"（世界知识）

一个词的概率背后，是整个语言体系、物理世界和人类知识的投影。当模型为了"猜得更准"而不断调整参数时，它被迫学到了这一整套隐含的知识结构。

这就像你在《原神》里打周本BOSS。你的目标很简单——把BOSS的血条打到零。但这个"简单"的目标逼迫你学会了：BOSS每个阶段的出招规律、元素反应的最佳搭配、队伍轮换的时机、走位的最优策略。你只是"打血条"，但你学会的是整个战斗系统。

### 损失函数和训练过程

具体到数学上，预训练用的是**交叉熵损失**：

模型对"下一个词可能是哪个"输出一个概率分布（词表中每个词都有一个概率值）。然后把模型输出的概率分布，跟"真实的词"（one-hot向量，只有正确答案是1，其他都是0）做交叉熵。差距越小，损失越小。

每个训练步骤大致如下：

1. 从数据中取一个批次（batch）的文本，比如128段。
2. 每段切成固定长度（比如4096个token）。
3. 对于每段的每个位置，让模型预测下一个token。
4. 计算所有位置上预测值和真实值的交叉熵损失。
5. 反向传播，更新模型参数。
6. 重复步骤1-5，直到模型在"没见过的新数据"上的损失不再下降。

这个过程通常需要几周到几个月，在几块到几万块GPU上并行运行。中间任何一个环节出问题（某块GPU挂了、数据中心断电、代码有bug），都可能需要从上次的存档点重启——现代LLM训练每隔几小时存一次检查点（checkpoint），就像你在玩《艾尔登法环》时必须定期在赐福点坐下存档。

### 扩增法则——"越大越好"是有数学依据的

2020年，OpenAI的研究员发现了一个后来被称为**扩增法则**（Scaling Laws）的规律：对语言模型来说，**测试损失**与**模型大小、数据量、计算量**之间，遵循一个清晰而稳定的幂律关系。

用人话翻译：你把模型参数量翻倍，损失会按一个可预测的比例降低。你把数据量翻倍，损失也会按可预测的比例降低。你把计算量翻倍，结果也一样。

这听起来很"废话"——模型越大当然越强。但扩增法则的意义不在于说"越大越好"，而在于说"好多少是可以提前算出来的"。你可以在训一个万亿参数模型之前，先用小模型测出各项关系系数，然后精确估算"如果要达到某个损失水平，最优的模型大小、数据量和计算量配比是多少"。

不过2023-2024年又出现了一个新发现：按扩增法则算出的"最优"配置，实际上**数据量才是瓶颈，不是模型大小**。换句话说，很多大模型其实"没吃饱"——数据量配不上它们的参数量。于是出现了像LLaMA这样的模型：用"相对较小"的参数（70亿-650亿，在当时算小），配以"极其充分"的数据训练（2TB），效果反超了参数量大得多但数据没喂够的模型。

对此，中国的DeepSeek团队在2024年用DeepSeek-V3给出了一个漂亮的示范：6710亿参数（但每次推理只用370亿，因为用了MoE架构），数据喂得极其充分（14.8TB高质量多语言数据），配合高效的训练工程（训练成本仅约557万美元），最终在多语言理解和代码生成等核心指标上追平甚至超越了同期最顶尖的闭源模型。

### 中国大模型的预训练版图

在全球LLM预训练格局中，中国团队已经形成了"一超多强"的局面。以下是2024-2025年的国产主力模型：

| 模型 | 开发团队 | 参数量 | 核心特色 |
|------|---------|--------|---------|
| **DeepSeek-V3** | 深度求索 | 671B (MoE, 激活37B) | 极致性价比，开源，训练成本仅~557万美元，性能追平GPT-4o |
| **通义千问 Qwen 2.5** | 阿里云 | 0.5B-72B多尺寸 | 中文最强之一，全面开源，生态最丰富 |
| **文心一言 4.0** | 百度 | 未公开（估计数千亿） | 搜索增强，中文理解深度出色 |
| **ChatGLM-4** | 智谱AI/清华 | 130B左右 | 中英双语均衡，学术基因浓厚 |
| **混元 Large** | 腾讯 | 389B (MoE) | 腾讯生态内深度整合 |
| **Yi-Lightning** | 零一万物 | 未公开 | 轻量高性能路线，推理速度极快 |
| **MiniMax-01** | MiniMax | 456B | 超长上下文（100万token），国内首批 |

国产LLM的崛起路径和GPT系列不同。GPT系列基本上是"一个模型打天下"——一个超大模型覆盖所有语言和任务。国产模型则走了"多线并进"的路：

- **DeepSeek走"极致性价比+开源"路线**：用MoE架构大幅降低推理成本，API价格仅为GPT-4o的几十分之一，同时完全开源模型权重，让中小企业和研究者能在本地部署。
- **Qwen走"全尺寸覆盖+生态"路线**：从0.5B到72B，从手机端到服务器端，全部开源。社区基于Qwen衍生出了几千个微调版本。
- **文心一言走"搜索增强"路线**：背靠百度搜索引擎，在事实性和时效性上有天然优势。
- **ChatGLM走"学术驱动"路线**：源自清华KEG实验室，论文产出量极高，对学术界友好。

这一局面就像《王者荣耀》的职业联赛：GPT目前相当于KPL的全冠王，但国产模型已经在特定赛道上练出了独门绝技——DeepSeek的成本控制、Qwen的中文能力和开源生态、文心的搜索增强，都是GPT难以替代的优势。

## 🎮 类比理解

LLM预训练就像你在《魔兽世界》里练一个"知识型"角色：

- **收集数据 = 跑图刷怪**：你不能只在一个区域刷——要去艾尔文森林（英文维基）、贫瘠之地（中文百科）、荆棘谷（GitHub代码库）、纳格兰（学术论文库）。每个区域掉落的"经验值"类型不同，你需要全部刷到。
- **数据清洗 = 分解装备**：打到的垃圾装备（低质量文本）直接分解或者扔掉，只保留有用的。同样的装备（重复内容）只留一件，多了占背包（算力）。
- **分词器 = 背包整理插件**：把乱七八糟的掉落物品自动分类、打标签、压缩存放——大件物品占一格（"中国"是一个token），小零件打包（生僻词被切成字符级token）。
- **预训练 = 挂机刷经验**：你的角色站在训练场里，系统不断给你看"一段文字→下一个词"，你做错了就挨罚（损失增加），做对了就奖励（损失减少）。好几周之后（几千块GPU跑几周），你升到了满级——现在你可以"续写任何句子"。
- **Scaling Law = 升级指南**：你精确知道"升一级需要多少经验值、在当前地图刷怪每小时的效率是多少"，于是能提前算出"到达60级需要200小时"——不会盲目瞎练。

## 💡 本章彩蛋

GPT-3在2020年刚发布时，很多人都对它的2万多个英伟达V100 GPU的训练集群感到震惊——这些GPU如果用来挖矿，一天就能赚几十万美元。但OpenAI选择了"烧钱造大脑"。

然而到了2024年，DeepSeek-V3的亮相彻底刷新了人们的认知：671B参数的模型，比GPT-4o的1.76T参数少得多，但在Open LLM Leaderboard上，它在多个benchmark上的得分已经非常接近GPT-4o——而训练成本仅为557万美元。这是什么概念？GPT-4的训练成本据估计在6300万到1亿美元之间。DeepSeek用了不到十分之一的算力，做出了接近的性能。这在全球LLM圈引发了巨大震动——被称为"DeepSeek时刻"。

另外，一个关于数据的冷知识：Common Crawl（公共网络爬虫档案）是绝大多数LLM的数据源——它每个月自动爬取一次全网网页并存档。截至2024年，Common Crawl已经积累了超过1000亿个网页、10PB以上的数据。你的博客、你在论坛的回复、你写的开源文档——如果它们能公开访问，大概率都被LLM"读过"。某种意义上说，每个在互联网上留下文字的人，都参与了这场"造脑运动"。

中文模型还有一个特别的挑战：中文互联网的高质量文本，无论在绝对数量还是类型多样性上，都远远不如英文互联网。很多国产团队的做法是：先用英文数据训练"通用语言能力"，再用高质量中文数据做"强化中文能力"——有点像你先用英语学会了游泳，然后换到中文游泳池再练几圈。这条路被Qwen和DeepSeek证明是行得通的。
